Я хочу приспособить Glide на macOS под очень узкий сценарий для qutebrowser.

Мне не нужен полноценный tiling window manager и не нужен общий window management для всей системы.

Нужен только такой минимальный workflow:
	•	в qutebrowser я нажимаю Cmd-Enter
	•	текущее окно qutebrowser автоматически делится на два окна рядом (side-by-side)
	•	во втором окне я могу открыть другую страницу и что-то быстро сделать
	•	потом я закрываю второе окно
	•	первое окно qutebrowser должно автоматически снова занять всю доступную ширину/область, как было до split

Также нужен минимум навигации:
	•	hotkey для перехода между левым и правым окном
	•	желательно только для этих двух qutebrowser окон в рамках этого сценария

Очень важно:
	•	Glide не должен заметно вмешиваться в остальную систему
	•	он не должен управлять всеми окнами подряд
	•	он должен работать только в этом сценарии, только для qutebrowser, только для временного split
	•	всё остальное лишнее

Что нужно от решения:
	1.	Понять, можно ли вообще так ограничить Glide под этот узкий сценарий.
	2.	Если да:
	•	дать минимальную рабочую конфигурацию
	•	дать готовые файлы целиком
	•	указать пути файлов перед кодом
	3.	Если Glide для этого неудобен или не подходит, предложить более простой вариант, но всё ещё под qutebrowser на macOS.
	4.	Нужен именно автоматизированный сценарий, а не ручной split через hotkeys/window manager команды.
	5.	Я использую fish shell и macOS.
	6.	Не использовать Python, если можно обойтись без него.
	7.	Комментарии в коде — только на английском.

Итоговая цель:
получить максимально простой “temporary browser split” для qutebrowser:
	•	вызвать split
	•	поработать во втором окне
	•	закрыть второе окно
	•	вернуться к одному окну на всю ширину
